## Decisions (and why)
- **Three domains** (schema/data/mechanism). The render mechanism was already outside the vault; the remaining schema+data co-location is Vault-LD by design. Folder layout is cosmetic because IRIs/edges are name-based — so segregation is a **commit-discipline** problem (the hook), not a relocation.
- **`validation/` over `shapes/`**: name the purpose, not the SHACL mechanism; also removes the near-collision with the spec-fixed `Ontologies/`.
- **`_schema`/`_data`** (underscore = "partition, not content"): mirrors the exporter's own `schema.ttl`/`data.ttl` output split.
- **Kept `rg`**: a CURIE prefix is a provenance handle, not a domain descriptor; and "resume" wouldn't fix the objection (a résumé is itself a representation of the career, same as the graph). Domain-honest target would be `career`, but YAGNI.
- **RoleFraming = note, not inline override**: exporter has no blank nodes; keying on `rg:audience` (not an employer) keeps reusable framings public. Projection uses `DELETE`/`INSERT` (surgical) over `CONSTRUCT`, `initBindings` (injection-safe), and fail-fast conflict detection. Honesty invariant: default `roleName` stays truthful; a framing is an alternative honest phrasing.
- **Skills taxonomy**: ESCO is the correct primary anchor (official + SKOS/RDF-native + EU) — already in the model; finish mappings in M6.
