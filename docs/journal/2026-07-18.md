## Decisions (and why)
- **`rg:lastActivity` as a first-class property** (not prose, not `endDate` reuse): recency must be queryable to tell the "still doing DE work last month" currency story; reusing `endDate` would conflate "project finished" with "last touched". Hand-captured snapshot, marked with capture date — keeps the build offline / vault-as-truth; CI refresh is a future option.
- **Athena + AWS IAM deliberately left as no-evidence stubs**: rag-bench-analytics' warehouse is RDS Postgres, not Athena; nothing gives IAM explicit evidence. Promoting them would violate the project's own anti-keyword-stuffing thesis. This is the differentiator working as designed.
- **One note per repo (not grouped)** + **mint all approved net-new skills**: owner's calls via AskUserQuestion. Course exercises (Udacity/EAE) modeled at honest `aware` level rather than inflated.
- **`Data Engineering Zoomcamp Coursework`** named distinctly from the same-named *cert* note (name-based identity → IRI collision). Cert *certifies* the skills; the coursework repo *evidences* them (a cert can't satisfy the evidence rule).
- **Test exemplar `Terraform` → `Amazon Athena`**: `test_unclaimed_stub_skill_is_exempt` needs a genuinely unreferenced stub; portfolio projects now claim Terraform via `usedSkill`.
- **Committed by domain, not by session**: shared files (`shapes.ttl`, `context.jsonld`, `test_validate.py`) carry both this session's and the interview's edits; file-level separation is impossible and hunk-staging is interactive/blocked. Three domain-pure commits satisfy the hook and are the only clean split available.
