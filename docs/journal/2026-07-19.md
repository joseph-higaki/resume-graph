## Decisions (and why)
- **One shared `resume_model.py`, three dumb renderers**: keeps SPARQL/rdflib knowledge in exactly one place; JSON/PDF/HTML exporters can't drift on how a position or skill is shaped.
- **Vanilla canvas, no framework, for graph.html**: this is a preview artifact meant to open directly from disk before the real Astro/Sigma site exists ‚Äî zero build step, zero external fetch, works over `file://`.
- **No browser in this environment** ‚Üí verified graph.html via a headless harness (Node + DOM/canvas stubs) running the actual emitted JS, then dumped post-simulation node positions to matplotlib to visually confirm cluster quality (Data Eng, AI Eng, Software Eng, Delivery, Cloud/AWS/GCP regions, Person hub central) instead of trusting the code by inspection alone.
- **Palette self-correction**: initial custom Position color (#22a06b) sat too close to the Skill color and Positions link heavily to Skills ‚Äî reassigned Position‚Üíviolet, Education‚Üígreen using the validated categorical palette with graph-aware placement (confusable green pushed to peripheral Education).
- **Explicitly stopped short of M4/M5**: user's instruction was "do not end2end ui build" ‚Äî graph.html is a preview to be tweaked, not the production site.
