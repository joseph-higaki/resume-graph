## Decisions (and why)
- **Vault folder names `Ontologies/rg/` and `Vocabularies/SkillCategories/`** (not the brief's `ontology/` / `categories/`): the vendored exporter classifies the schema layer *by those exact folder names* (Vault-LD SPEC §3). Lowercase/renamed folders would silently demote the ontology to instance data. Updated `.claude/CLAUDE.md` to match.
- **Bullets are typed notes in `vault/bullets/`**, not inline body annotations: Vault-LD never exports the body (§5.3) and the exporter has no blank-node support, so a `[[Bullet]]` note (`text`/`audience`/`bulletOf`/`order`) is the only carrier that reaches the graph. Documented in the brief per its "decide in M1" instruction.
- **`domainIncludes`/`rangeIncludes` over `rdfs:domain`/`range`** on properties: multiple `rdfs:domain` values entail class *intersection* (a subject inferred to be all of them at once); schema.org's variant documents a union without that entailment.
- **Evidence rule as a `sh:sparql` constraint**: "claimed" is a disjunction (`rg:level` present OR referenced via `rg:usedSkill`) that plain SHACL property shapes can't express. Tests cover both trigger directions plus the demanded-stub exemption.
- **ESCO absence as `sh:Warning`** + pySHACL `allow_warnings=True`: reports on every run, never fails the gate; advisory until M6.
- **Amended the root commit** instead of adding a new one, because the repo had never been pushed — cleanest way to keep the gitignored `CLAUDE.local.md` out of public history.
- **Split `export`/`project`/`site` Makefile targets**: a grouped `export project site:` line is parsed as GNU make's `export` *directive*, not a rule.
